# Stage 4 Acceptance

## 范围

Stage 4 完成附件与版本历史：

- `attachments` 表。
- `document_versions` 表。
- 文档创建、保存和回滚后生成版本快照。
- 编辑器展示版本历史并支持回滚和删除。
- 编辑器支持附件上传、下载和删除。
- 后端对附件路径做归一化校验，下载和删除前重新校验文档所有权。

## 验收步骤

1. 启动 MySQL 和 Redis。
2. 启动后端，确认 Flyway 执行 `V5__create_attachment_version_tables.sql`。
3. 登录前端并创建知识库。
4. 创建一篇文档，确认版本历史出现版本 1。
5. 修改文档并保存，确认版本历史新增版本。
6. 回滚到旧版本，确认编辑器标题、摘要、状态和正文恢复。
7. 删除一个不再需要的历史版本，确认版本历史列表同步减少。
8. 上传一个附件，确认附件列表显示文件名、大小和类型。
9. 下载附件，确认文件内容可保存。
10. 删除附件，确认附件列表同步更新。

## 学习重点

- 版本历史保存的是完整快照，不依赖当前文档内容。
- 回滚也是一次写操作，应继续生成新版本，保留审计链路。
- 版本删除只用于清理历史快照，不影响当前文档内容。
- 附件表保存元数据，文件内容放在可配置存储目录。
- 文件名不能直接作为磁盘路径，必须生成服务端安全文件名并校验最终路径位于附件根目录内。
